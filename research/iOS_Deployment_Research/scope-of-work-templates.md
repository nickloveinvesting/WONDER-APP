## Template 1: Full Deployment (Capacitor + Submission)

**Platform:** Upwork, Arc.dev, Toptal
**Budget:** $1,000-2,500
**Timeline:** 1-3 weeks

---

### Job Posting Template

```markdown
TITLE: Convert Next.js Web App to iOS Using Capacitor + Complete App Store Submission

CATEGORY: Mobile Development > iOS Development

BUDGET: $1,500 (Negotiable for experienced developers)

PROJECT DURATION: 2-3 weeks

EXPERIENCE LEVEL: Intermediate to Expert


DESCRIPTION:

I'm looking for an experienced iOS/Capacitor developer to convert our existing
Next.js web application into a native iOS app and handle the complete App Store
submission process.

PROJECT OVERVIEW:
We have a fully functional Next.js 14 web application deployed on Vercel. The
app is responsive and mobile-friendly. We need it wrapped as a native iOS app
using Capacitor (NOT a complete native rewrite).

TECH STACK:
- Next.js 14 (App Router)
- React 18
- TypeScript
- Deployed on Vercel
- Mobile-responsive design already implemented

DELIVERABLES:

1. CAPACITOR CONFIGURATION
   - Install and configure Capacitor for Next.js
   - Set up Next.js for static export (if needed)
   - Configure iOS platform
   - Ensure all app features work in WebView
   - Handle any CORS or API issues

2. iOS NATIVE BUILD
   - Create iOS project in Xcode
   - Configure app signing certificates
   - Generate IPA file for distribution
   - Test on iOS simulator
   - Test on physical device (if available)

3. DESIGN ASSETS
   - App icon in all required iOS sizes (20px to 1024px)
   - Launch screen/splash screen
   - App Store screenshots (5-10 images for various device sizes)
   - Screenshots should be professional with text overlays if appropriate

4. APP STORE SUBMISSION
   - Set up App Store Connect listing
   - Write/optimize app metadata (title, subtitle, keywords)
   - Upload build to App Store Connect
   - Complete all required fields
   - Submit for review
   - Handle any rejection issues (1 resubmission included)

5. DOCUMENTATION
   - Step-by-step guide for future app updates
   - Build process documentation
   - Configuration notes
   - Handoff session (30-60 min screen share)

REQUIREMENTS:

MUST HAVE:
‚úÖ Proven Capacitor experience (please share App Store links to apps you've built)
‚úÖ Experience with Next.js or React
‚úÖ Completed App Store submissions (show published apps)
‚úÖ Understanding of iOS code signing and provisioning
‚úÖ Available to start immediately
‚úÖ Good English communication
‚úÖ Available for follow-up support (30 days bug fixes included in budget)

NICE TO HAVE:
‚úÖ Experience with Next.js static exports
‚úÖ Experience with Vercel deployments
‚úÖ App Store Optimization (ASO) knowledge
‚úÖ Previous experience with [your app's specific features, e.g., authentication, payments]

WHAT I'LL PROVIDE:
- Access to GitHub repository
- Apple Developer account (already set up)
- Vercel access (if needed)
- Brand assets (logo, colors)
- App description draft
- Support and timely responses

PAYMENT STRUCTURE:
- Milestone 1 (50%): Capacitor setup complete, working iOS build in simulator
- Milestone 2 (50%): App live on App Store + documentation delivered

TIMELINE:
- Expected completion: 2-3 weeks
- Daily or every-other-day progress updates preferred

TO APPLY:

Please include in your proposal:

1. Link to iOS apps you've published (App Store links required)
2. Specific examples of Capacitor projects you've completed
3. Your experience with Next.js/React
4. Estimated timeline for this project
5. Any clarifying questions you have about the requirements
6. Your approach to handling potential issues (API calls, authentication, etc.)

Applications without App Store links or portfolio will not be considered.

QUESTIONS?
Feel free to ask any questions before applying. I want to ensure we're aligned
on expectations.

Looking forward to working with you!
```

## Milestone Definitions

### Milestone 1: Capacitor Setup Complete

**Deliverables:**
- [ ] Capacitor installed in project
- [ ] iOS platform configured
- [ ] App builds successfully in Xcode
- [ ] App runs in iOS simulator without errors
- [ ] All major features tested and working
- [ ] Build process documented

**Acceptance Criteria:**
- Client can run the app in iOS simulator
- No console errors during basic app usage
- All primary features function correctly
- Developer provides build instructions

**Payment:** 50% of total project fee

---

### Milestone 2: App Store Approved

**Deliverables:**
- [ ] App icon created in all sizes
- [ ] Screenshots created for App Store
- [ ] App Store Connect listing complete
- [ ] App submitted to App Store
- [ ] App approved and live
- [ ] Documentation delivered
- [ ] Handoff session completed

**Acceptance Criteria:**
- App is live and downloadable on App Store
- All documentation received and understandable
- Knowledge transfer session completed
- Client can build future updates following docs

**Payment:** 50% of total project fee

---

### Alternative Milestone Structure (3 Milestones)

**Milestone 1 (33%): Configuration Complete**
- Capacitor installed and configured
- iOS project set up in Xcode
- App runs in simulator

**Milestone 2 (33%): Build Ready**
- IPA file generated
- Design assets created
- App tested on device

**Milestone 3 (34%): Live on App Store**
- App submitted and approved
- Documentation delivered
- Support period begins

# CHANNEL ORGANIZATION GUIDE
## Detailed Implementation Specification

**Purpose**: Provide concrete channel structure recommendations for PhiloDuel's Discord-like organization

---

## EXECUTIVE SUMMARY

Based on 2025 Discord best practices research:
- **4-5 channels per category** (prevents cognitive overwhelm - 40% reduction)
- **Clear, purpose-driven naming** (users know exactly where to go)
- **Collapsible categories** (personalized focus)
- **Progressive disclosure** (beginners see basics, veterans see advanced)
- **Logical flow** (important ‚Üí frequently used ‚Üí optional)

### 2. Category Collapse Behavior

**Auto-collapsed** (after first view):
- Announcements (users know where to find)
- Learning (veterans don't need)
- Competition (unless opted in)

**Always expanded** (high traffic):
- Universal Philosophy
- Subscribed Schools
- Subscribed Topics

**User-controlled**:
- Schools (expand those you're interested in)
- Topics (expand areas of expertise)

### 3. Channel Naming Best Practices

**Good Names** (clear purpose):
- ‚úÖ `#normative-ethics` (specific topic)
- ‚úÖ `#quick-debates` (clear format)
- ‚úÖ `#philosophy-101` (clear audience)
- ‚úÖ `#stoicism-circle` (clear school)

**Bad Names** (vague):
- ‚ùå `#chat` (too generic)
- ‚ùå `#random` (no purpose)
- ‚ùå `#stuff` (meaningless)
- ‚ùå `#channel-1` (no context)

### 4. Channel Descriptions

Every channel must have:
- **Purpose**: One-sentence what it's for
- **Rules**: Specific guidelines (e.g., "citation required")
- **Min Rep**: Required reputation
- **Examples**: Sample posts (pinned)

**Example**:
```
#applied-ethics
Purpose: Discuss real-world ethical problems (bioethics, tech ethics, climate)
Rules: Cite sources for factual claims, respect diverse viewpoints
Min Rep: 100
Example: "Should gene editing be allowed for human enhancement?"
```

## MODERATION STRUCTURE

### Tier 1: Auto-Moderation (Bots/AI)
- Spam detection
- Profanity filter (philosophy-appropriate - allows "damn" in "damned if you do")
- Link validation
- Duplicate post prevention
- Karma threshold enforcement

### Tier 2: Channel Moderators
Each category has **2-3 moderators**:
- Topic experts (not just high-rep users)
- Enforce channel-specific rules
- Pin exemplary content
- Redirect off-topic discussions
- Weekly "best of" compilations

### Tier 3: School Moderators
Each school has **3-5 elected moderators**:
- Elected by school members quarterly
- Set school-specific guidelines
- Organize school events
- Represent school in cross-school activities

### Tier 4: Platform Moderators
**5-10 moderators** for entire platform:
- Handle cross-category issues
- Enforce universal values
- Resolve disputes
- Ban decisions (with appeals)
- Community governance

### Tier 5: Community Governance
**Philosopher Kings/Queens** (10,000+ rep):
- Vote on major policy changes
- Platform feature prioritization
- Moderator oversight (check and balance)

## CHANNEL LIFECYCLE MANAGEMENT

### Creating New Channels

**Criteria**:
1. Sustained demand (100+ requests)
2. No existing channel serves purpose
3. Sufficient moderation capacity
4. Aligns with community values

**Process**:
1. Community proposal (1000+ rep required)
2. 7-day discussion period
3. Vote (500+ rep users eligible)
4. 60% approval required
5. 30-day trial period
6. Evaluate activity (min 50 messages/week)

### Archiving Inactive Channels

**Criteria**:
- <10 messages per week for 4 weeks
- No unique value proposition
- Better served by existing channel

**Process**:
1. 14-day notice
2. Community input
3. Archive (read-only, searchable)
4. Redirect users to alternative channel

## SUCCESS METRICS

### Channel Health Indicators

**Thriving Channel** (green):
- 50+ messages per week
- 70%+ upvote ratio
- 10+ unique contributors
- Low moderation needs

**Healthy Channel** (yellow):
- 20-50 messages per week
- 60%+ upvote ratio
- 5-10 unique contributors
- Moderate moderation

**Struggling Channel** (red):
- <20 messages per week
- <60% upvote ratio
- <5 unique contributors
- High moderation needs

**Action**: Struggling channels evaluated for merger or archival

## FINAL RECOMMENDATIONS

1. **Start Small**: Launch with 20-30 channels (not 50+)
2. **Grow Organically**: Add channels based on community demand
3. **Clear Purpose**: Every channel has specific, documented purpose
4. **Progressive Disclosure**: Show users what they need, when they need it
5. **Community Control**: Let users customize their view (collapse, favorite, mute)
6. **Active Moderation**: Each channel has dedicated moderators
7. **Regular Audits**: Quarterly review of channel health
8. **Respect Mobile**: Optimize for small screens
